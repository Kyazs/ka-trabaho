## Objective

Validate and refine the existing backend job recommendation API. Ensure the `/api/job-recommendation` endpoint returns correct, enriched data and handles edge cases properly.

Purpose: The backend is the foundation for the job matching feature; it must reliably map AI job suggestions to real TESDA job data including salary, demand, and required courses.

Output: Tested and verified API endpoint with correct error handling and data enrichment.

### Key Interfaces

From server.ts (already implemented):
- `POST /api/job-recommendation` - accepts profile data, returns enriched job recommendations
- `mapAiJobsToExistingData(aiJobs, regionCode)` - maps AI job titles to real SECTORS_DATA
- `FALLBACK_JOB_RECOMMENDATION` - static fallback when AI fails

From src/data/tesdaData.ts:
- `SECTORS_DATA` - array of sectors with jobs and courses
- `JobRole` - type with title, averageSalary, demandLevel, mappedCourses
- `TesdaCourse` - type with code, name, duration, level

| T-01-04 | Information Disclosure | AI prompt | accept | No PII beyond age/region |
| T-01-05 | Denial of Service | API endpoint | accept | No rate limiting implemented |
