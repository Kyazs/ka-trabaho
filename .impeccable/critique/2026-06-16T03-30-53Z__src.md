## Anti-Patterns Verdict

**LLM assessment**: Partial AI tell detected. No side-stripe borders, gradient text, glassmorphism, or section eyebrows. But three tells remain:
1. Hero-metric template on landing stats section (big numbers + small labels + blue/gold container)
2. Ghosted numbered markers on How-it-Works cards (giant faded numbers — the AI reflex even though steps ARE a sequence)
3. Identical card grids across 6 different sections with no variation in shape, size, or emphasis

**Deterministic scan**: Zero findings across all src files. Codebase is clean of detectable anti-patterns.

## Overall Impression

The app is technically competent and culturally authentic — the Taglish voice is genuinely well-executed, the privacy architecture is structurally coherent, and the "I don't know" escape hatch on regions shows real audience understanding. But the landing page is too long and too redundant for a 3G mobile audience, the font-display usage violates the product register ban on display fonts in UI labels/buttons/data, and the wizard progress indicator counts system steps (processing, results) as user steps, making the form feel longer than it is. The single biggest opportunity: trim the landing page by ~35% and fix font-display misuse — both changes serve the product register (design serves usability) with minimal risk.

## What's Working

1. **Taglish that's actually Taglish** — the bilingual system is two registers of the same voice, not a toggle between separate languages. Chat greeting, wizard skip text, and error fallbacks are natural Filipino constructions with embedded English loan-words. Rare and genuinely well-executed.

2. **"I don't know" escape hatch on region selection** — for an audience that may not know their administrative region code, this single option prevents a dead end. Combined with optgroup island classification, it demonstrates real understanding of the audience's knowledge gaps.

3. **Privacy-first architecture with honest messaging** — "Nothing is uploaded to any server" is prominent, bold-highlighted, and structurally coherent with the localStorage-only persistence model. "Clear My Data" gives shared-device users a concrete exit.

## Priority Issues

### P1 — Hero mock card simulates real results before user provides data
**What**: LandingPage hero right panel shows pre-selected chips + blinking cursor + "97% Match" badge for a specific course.
**Why it matters**: Maria (skeptical of gov portals) may think the app already found her a match — so why fill out a form? Or worse: results are predetermined/fake. Undermines trust at the exact moment it needs to build it.
**Fix**: Replace mock match result with an empty-state preview — show the question and chips, but replace the "97% Match" card with a faded placeholder ("Your match will appear here") or a simple pathway graphic (→ Assessment → Results).
**Suggested command**: /impeccable shape

### P1 — font-display (Plus Jakarta Sans) used in UI labels, buttons, and data
**What**: font-display appears on ~60% of non-heading text — buttons ("I-add", "Sunod"), labels ("Pumili ng Sektor"), data values (match scores, course codes, salary).
**Why it matters**: Product register bans "Display fonts in UI labels, buttons, data." Plus Jakarta Sans at 700 in 13–14px creates visual density that reads "designed" not "usable." Inter at those sizes is more readable and familiar.
**Fix**: Keep font-display on h1/h2/h3 and logo only. Switch everything else to font-sans (Inter): buttons, labels, tag text, data values, form section titles, card metadata.
**Suggested command**: /impeccable typeset

### P2 — Landing page excessively long for mobile-first 3G audience
**What**: 6 major sections (hero, trust strip, how-it-works, features, stats, CTA) = ~4,000+ px scroll on mobile. Features repeat trust strip content. Stats repeat trust strip figures.
**Why it matters**: More content = more loading, more scrolling, more cognitive processing before primary CTA. 3G prepaid data makes every KB count.
**Fix**: Remove stats section (trust strip already shows key figures). Reduce features from 5 to 3 core differentiators. Move remaining features to compact "Also includes" row. Cuts landing page ~35%.
**Suggested command**: /impeccable distill

### P2 — Wizard shows 7 steps when user only controls 4
**What**: stepOrder has 7 entries including processing and results (system states). Progress indicator counts all 7. "Step 5 of 7" at review step makes form feel 75% complete when user has done all the work.
**Why it matters**: "7 steps" is intimidating before starting. Progress bar jumps when system steps are added.
**Fix**: Show only 4 navigable steps: Profile → Interests → Skills → Goal. Processing/results are outcomes, not steps. Merge review into goal step as collapsible summary above submit.
**Suggested command**: /impeccable layout

### P3 — Inconsistent border-radius vocabulary
**What**: 6+ radius values without semantic mapping: rounded-xl, rounded-2xl, rounded-3xl, rounded-[10px], rounded-[20px], rounded-full. Hero card uses rounded-[20px] while similar cards use rounded-2xl.
**Why it matters**: Inconsistent radius creates subtle visual dissonance — cards that should feel related look slightly different.
**Fix**: Enforce DESIGN.md scale: pills=full, inputs=xl (12px), buttons=xl (12px), cards=2xl (16px), hero/large=3xl (24px). Remove all rounded-[Npx] custom values.
**Suggested command**: /impeccable polish

## Persona Red Flags

### Maria (19, out-of-school Filipino youth, shared basic smartphone, Taglish, skeptical of gov portals, 3G)
- Hero "97% Match" mock card — she thinks results are predetermined, undermining assessment value
- Privacy notice as dismissable banner — disappears after "Naiintindihan" and never reappears; shared-device risk
- Education dropdown English-only in Filipino mode — "Junior High School Graduate" regardless of lang
- "Assessment" in CTA text — sounds school-like, the institution she left. "Hanapin ang Kurso para sa Akin" would be more inviting

### Jordan (Confused First-Timer)
- No onboarding walkthrough — lands on content-heavy page with no "Start here" guidance beyond hero CTA
- "AI Match" tab label — doesn't know what "AI Match" means. "Find My Course" would be clearer
- Match score unexplained — "97% Match" could be interpreted as job guarantee or pass/fail grade
- No contextual help in wizard steps — no "Why do we need this?" explanation for each question

### Casey (Distracted Mobile User)
- No UI cancel button during AI processing — AbortController exists internally but no visible cancel button; stuck watching spinner
- Input text (interestInput, skillInput, chatInput) not persisted — lost on interruption, must retype
- Chat has no new-message indicator — no badge when AI replies while on another tab

## Minor Observations

1. Navbar `backdrop-blur-xl` — GPU-intensive on basic smartphones; consider `backdrop-blur-sm` + `bg-white/95`
2. `card-hover` translateY(-4px) on hover — hover doesn't exist on touch devices; pure desktop decoration
3. Chat character counter at `absolute right-2 bottom-1` — overlaps typed text on narrow screens
4. `animate-fade-in` 0.5s delay on every tab transition — compounds perceived loading on 3G; reduce to 0.15s
5. Error cards use hardcoded Tailwind red instead of kt-danger tokens — violates token system in spirit
6. `stagger-1` through `stagger-4` classes set `opacity: 0` but are never used — dead CSS risk
7. Bottom nav `h-16` + main `pb-24` — 8px more padding than needed; should be `pb-[68px]`
8. Demand-level badges always in English ("Very High", "High") regardless of lang setting

## Questions to Consider

1. If Maria can't read English well, why is the education dropdown only in English?
2. The "97% Match" mock simulates personalization before the user provides data — what happens to trust when the real result is 73%?
3. If the app is truly for shared devices on 3G, why does it load a ~1,600-line monolithic App.tsx with 30+ useState hooks?
4. The wizard asks for age 15–24 with no explanation of why. If someone is 25 or 14, the slider silently excludes them. Is this gatekeeping the North Star rejects?
