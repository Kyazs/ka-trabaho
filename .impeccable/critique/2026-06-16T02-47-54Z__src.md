## Anti-Patterns Verdict

**LLM assessment**: PASS. The app does NOT look AI-generated. The flag-anchored palette, bilingual voice, Rest-Flat Rule, and progressive disclosure are distinctive. No gradient text, no side-stripe borders, no ubiquitous eyebrows, no identical card grids. The landing page has a faint "template" aroma (4 section eyebrows, stats bar, watermark step numbers) but the rest of the app is clean and culturally authentic.

**Deterministic scan**: Clean — 0 findings across all 9 source files.

## Overall Impression

The app has matured significantly since the first critique (27 → 31 → 28). The drop from 31 reflects honest scoring with tighter criteria, not regression. The biggest remaining gap is the data privacy/management void — a shared-device app for Filipino youth with no way to clear personal data is a trust problem. The second gap is the region selector: 17 flat options with no grouping is a cognitive wall at the first decision point. The third is the wizard's inflexibility: 5 mandatory steps with no skip, no partial re-match, and no step persistence across sessions.

## What's Working

1. **Bilingual architecture is genuinely excellent, not performant.** Every surface has fil/en — not just labels but the entire voice. Taglish reads naturally, entryReqs are per-course bilingual, education options use the Philippine system. This is bilingual-first, not a bolted-on toggle.

2. **Progressive disclosure is the app's strongest pattern.** Explorer collapses jobs/courses behind accordions. Quick tags show 4 + "more". Job form auto-collapses. Wizard chunks a complex form into single-focus steps. Rare discipline in AI-generated UI.

3. **Error resilience with real alternatives.** When AI fails, the app names a concrete next step: browse Explorer, visit a TESDA branch, check the FAQ. The 504 auto-retry is smart. Rate limits show reset time.

## Priority Issues

**[P0] No data privacy mechanism for shared-device users**: All personal data (career goals, interests, chat history, match results) persists in localStorage with NO way to clear it. No logout, no "clear my data", no privacy mode, no session timeout. Maya shares a phone with 3 family members — her career dreams and AI counseling conversations are permanently visible to anyone who opens the app. Fix: Add "Clear All My Data" with inline confirmation. Add "New Session" on chat header. Show brief privacy notice on first visit. Suggested command: `/impeccable harden`

**[P1] Region selector is a cognitive wall**: 17 Philippine regions in a flat `<select>` with no grouping, no search, no geographic cues. Defaults to R9 (Zamboanga Peninsula). A 16-year-old in BARMM may not know their region code. Fix: Use `<optgroup>` to group by island group (Luzon/Visayas/Mindanao). Default to NCR. Add "Hindi ko alam ang region ko" escape hatch. Suggested command: `/impeccable harden`

**[P1] Chat history has no management controls**: Messages accumulate endlessly with no delete, no clear, no management. All messages persist in localStorage and are sent as context (last 20) with every API call. Privacy risk on shared devices. Stale context poisons AI responses. Fix: Add "Clear Conversation" button in chat header. Cap localStorage at 50 messages. Suggested command: `/impeccable harden`

**[P2] Job result cards are information-dense walls**: Each card contains 8+ distinct info elements with nested visual layers. On a 320px phone, a single card can be 800+ px tall. Fix: Make cards two-state: collapsed (title + match + demand + salary + CTA) and expanded (quote, description, required courses). Default to collapsed. Suggested command: `/impeccable layout`

**[P3] Wizard step state not persisted**: Form values persist but currentStep does not. If Casey closes the browser during "interests" and returns, they restart at step 1 with pre-filled data but no indication of where they were. Fix: Persist `currentStep` to localStorage. Restore on mount. Suggested command: `/impeccable harden`

## Persona Red Flags

**Casey (Distracted Mobile — 3G, interrupted)**: Hero question card looks tappable but isn't (chips + "97% Match" badge have no interaction). No cancel button during processing despite AbortController in code. Step position lost on browser close/interrupt — must re-traverse 5 steps. Age slider hard to set precisely on small touchscreen with no tick marks.

**Jordan (Confused First-Timer)**: No explanation of what TESDA is on the landing page — assumes prior knowledge. "AI Assessment — Aktibo" label is jargon. Education options label Jordan's situation in deficit terms ("Drop-out"). No "I don't know" for region selector. Validation errors may appear below the fold on small screens.

**Sam (Accessibility-Dependent)**: FAQ question prefixes "T1"/"Q1" are meaningless auditory clutter for screen readers (no aria-hidden). Overflow menu in Navbar doesn't trap focus or support arrow-key navigation. Age slider visual value isn't programmatically associated with the input for dynamic announcements. Chat send button's disabled state has no aria-disabled.

**Maya (Shared-Device User)**: All chat history visible to anyone who opens the app — no clear button. Career goals and AI counseling persist permanently. No privacy notice on first visit. No "end session" or "clear data" anywhere. The only way to clear data is through browser settings — not discoverable for the target demographic.

## Minor Observations

- "TL" language label is non-standard; Filipino's ISO code is "fil", not Tagalog's "tl"
- "Simula" for Home in BottomNav is semantically unusual (Simula = start, Bahay = home)
- Education option value "Vocational College Undergraduate" doesn't match label "College Level Undergrad / Drop-out"
- `×` remove buttons on tags have 44px touch target but no visible boundary/hover state
- Footer appears on every page including chat, creating 200+ px of dead scroll below the chat input
- Landing hero conditional `{text(copy.heroTitleLine2) === "para sa" ? "sa " : ""}` is dead code in English mode
- No 404 handling beyond silent redirect to landing
- Gold (#FCD116) on white fails WCAG AA (1.95:1) but is only used on dark backgrounds — no actual violation
- Chat errors appearing as bot messages breaks the counselor persona and creates confusion

## Questions to Consider

- "What if the wizard didn't exist at all? The landing page already has a working concept: 6 chips + instant result. What if those chips were real — one tap and the AI returns results with defaults?"
- "What would happen if the AI server is down for a week? Explorer doesn't connect to the user's profile at all. A local heuristic matching interest keywords to sector names would deliver 70% of the value with 0% API dependency."
- "Why not make the profile exportable as a URL-encoded link or QR code? That solves shared-device privacy, persistence without accounts, and shareability all at once."
