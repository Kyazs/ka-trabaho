## Anti-Patterns Verdict

**LLM assessment**: The interface does NOT look AI-generated. The flag-anchored palette, bilingual voice, and "Open Doorstep" warmth are distinctive. No gradient text, no side-stripe borders, no identical card grids, no ubiquitous eyebrows. The Rest-Flat Rule is rigorously enforced. The progressive disclosure on Explorer/FAQ/Jobs is well-executed. The main tells are: (1) demand badge colors using Tailwind defaults (red-50/amber-50) instead of the kt- token system, and (2) the PageHeader accent system having 6 names mapping to 3 visual outcomes.

**Deterministic scan**: Clean — 0 findings across all 8 source files.

## Overall Impression

The app has made significant progress since the initial critique (27/40 → 31/40). The biggest wins are the Rest-Flat enforcement (22 shadows removed), progressive disclosure (Explorer/FAQ/Jobs restructured), and localStorage persistence. The remaining gap is between the design system's rigor on the surface (tokens, shadows, typography) and its gaps in the data layer (missing regions, untranslated entryReqs, non-tokenized demand colors). The single biggest opportunity: make the Explorer tab feel as guided as the wizard — right now it's the only tab without a clear primary action.

## What's Working

1. **Progressive disclosure execution**: The Explorer Jobs/Courses accordions, FAQ single-open accordion, Jobs form collapse, and tag chunking (≤4 + "show more") are all well-implemented with proper aria-expanded/controls and smooth chevron rotation.

2. **Rest-Flat Rule as doctrine**: Zero decorative resting shadows across all 5 component files. Hover-only shadows are consistent. This gives the interface a calm, professional feel that matches the "Open Doorstep" north star.

3. **Bilingual voice authenticity**: Taglish copy reads naturally, not mechanically translated. The confirm dialog ("Sigurado ka bang gusto mong simulan muli?") and status messages ("Handang tumulong ngayon") are idiomatic.

## Priority Issues

**[P1] Focus indicator invisible on white backgrounds**: The focus ring uses `--color-kt-blue-light` (#E8F0FE) which has ~1.3:1 contrast against white — failing WCAG 2.1 AA non-text contrast (3:1 minimum). Keyboard users cannot see focus. Fix: Change focus ring color to `--color-kt-blue` (#0F3D91) in index.css. Suggested command: `/impeccable harden`

**[P1] Chat messages not persisted**: Navigating away or refreshing loses the entire conversation including enrollment advice and document lists. All other profile data IS persisted. Fix: Serialize chatMessages to localStorage (convert Date timestamps to ISO strings, restore on mount). Suggested command: `/impeccable harden`

**[P1] Only 7 of 17 Philippine regions**: Users from 10 regions (R1, R2, R4B, R5, R8, R10, R12, R13, CAR, BARMM) cannot select their home region. This makes the app partially unusable for a large portion of the target audience. Fix: Add all 17 regions to tesdaData.ts. Suggested command: `/impeccable harden`

**[P1] Course entry requirements only in Filipino**: When lang === "en", entryReq still displays in Filipino ("Nakapagtapos ng Junior High School..."). Bilingual data is missing for a critical enrollment-eligibility field. Fix: Add English entryReq translations to tesdaData.ts. Suggested command: `/impeccable harden`

**[P2] Typing indicator not aria-hidden**: The animated dots and "Ka-TrabaHO is typing..." text in App.tsx:1092-1103 will be read aloud by screen readers as nonsensical content. Fix: Add `aria-hidden="true"` to the typing indicator wrapper. Suggested command: `/impeccable harden`

**[P2] Demand badge colors not tokenized**: `bg-red-50 text-red-700 border-red-200` and `bg-amber-50 text-amber-700 border-amber-200` use Tailwind defaults instead of the kt- token system. These are the last non-semantic color references. Fix: Add demand-level tokens or use semantic tokens. Suggested command: `/impeccable colorize`

**[P2] Selected skill tags inconsistent**: AssessmentWizard uses `bg-kt-blue text-white` for selected skills; ProfileMiniForm uses `bg-kt-gold text-kt-near-black`. Same interaction, different visual treatment. Fix: Standardize to one pattern (recommend blue for both, matching the DESIGN.md chip spec). Suggested command: `/impeccable polish`

**[P2] window.confirm() for Start Over**: The native confirm dialog is inaccessible — cannot be styled, traps focus, inconsistent screen reader behavior. Fix: Replace with an inline confirmation pattern (e.g., two-step click: first click shows "Are you sure?", second click confirms). Suggested command: `/impeccable harden`

## Persona Red Flags

**Casey (Distracted Mobile User)**: Chat messages lost on tab switch — if Casey switches to check a text message and the browser unloads the tab, the entire conversation including TESDA enrollment instructions disappears. The 5-tab bottom nav is slightly above the 4-choice guideline. The Explorer tab has no clear CTA to orient Casey on what to do first.

**Jordan (Confused First-Timer)**: The Explorer tab dumps Jordan into a search box + 5 sector buttons with no guidance. No tooltip or hint explains "pick a sector to see courses." The course entry requirements appear only in Filipino even when Jordan selected English, creating a comprehension barrier for English-mode users.

**Maya (Shared-Device User)**: Chat messages are not persisted. If Maya shares a phone and someone else closes the tab, her counseling conversation with specific TESDA center directions is gone. The rate limit badge shows "2/5 left" but doesn't say when it resets, so Maya can't plan her return visit.

## Minor Observations

- ProfileMiniForm uses `bg-kt-bg` instead of `bg-white`, making it visually recessed vs. all other cards
- FAQ answer text is `text-xs` (12px) on mobile — below the 14px body minimum from DESIGN.md
- Chat timestamps remain at `text-[11px]` (11px) — the last sub-12px text in the app
- Chat input character counter overlaps input text (absolute positioned, no right padding on input)
- `shimmer` CSS class defined in index.css but never used anywhere — dead code or unused opportunity
- BottomNav uses `role="tablist"`/`role="tab"` but no `role="tabpanel"` exists — misuses ARIA tabs pattern for navigation
- Landing page hero chips look tappable (pill shape, border) but aren't interactive — false affordance
- Step number decorations in landing page "How it works" are decorative but not `aria-hidden`
- `topSectors` field in RegionInfo is defined but never consumed by any component
- `navigate()` called during render body (App.tsx:82-84) — React anti-pattern, should be in useEffect
- Job form doesn't auto-collapse when results arrive — user must manually collapse
